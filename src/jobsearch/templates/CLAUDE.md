# career-hub — your professional life, centralized

This folder is the data root of a career-hub installation. It contains your
profiles, CVs, work experience, certificates, applications, and personal
brand assets, all organized so that any AI assistant can read your history
and help you produce job applications without you rewriting your story
every time.

## What lives here

| Folder | Purpose |
|--------|---------|
| `profiles/` | Your professional profiles (skills, narrative, fit rules). Each `PROFILE_*.md` has YAML frontmatter the CLI parses. |
| `cvs/` | CV templates (DOCX) by role and language. The CV-builder copies these and tailors them per application. |
| `roles/` | Optional. Role-family definitions and sector mappings if you target multiple verticals. |
| `work_experience/` | Your career history in STAR format with KPIs. **Single source of truth** the AI reads to personalize applications. |
| `documentation_hub/` | Past projects as reusable case studies (problem / approach / result / artifacts). The AI cites them in cover letters. |
| `certificates/` | Certifications and credentials with metadata (issuer, date, expiry). |
| `personal_brand/` | Brand statement, voice and tone, content strategy. The AI reads `voice_and_tone.md` to keep cover letters in your voice. |
| `applications/` | Generated by the CLI: one folder per company per application. |
| `jobsearch.db` | SQLite tracker for applications and scanned jobs. Do not edit by hand. |
| `portals.yml` | Companies and portals the scanner should query. |
| `config.toml` | Roles, sectors, CV mappings. Edit when you add or rename a role. |

## How to work with this directory using AI

If you opened this folder in Claude Code (or any AI agent that reads
`CLAUDE.md`), here is the contract:

1. **First time?** Ask the assistant: "explain this project and walk me
   through onboarding." It should describe the centralization vision,
   point you at the 3 levels in the upstream README, and offer to follow
   the project on GitHub
   ([@fernandosiavato](https://github.com/fernandosiavato)) and star the
   [repo](https://github.com/fernandosiavato/career-hub).
2. **Building your first profile?** Ask: "interview me to fill in
   `profiles/PROFILE_DATA.md`." The AI uses the YAML schema in the file
   itself as the contract.
3. **Applying to a job?** Run the `/apply` skill (Claude Code) with the
   JD pasted in chat. It produces a JD note, keywords + gaps file, cover
   letter, and a Python script that builds your tailored CV. You review
   and submit. Always.
4. **Filling in past experience?** Ask: "interview me about my role at
   <company>." Drop the STAR result into `work_experience/`.

## Privacy

Everything in this folder is **your** data. Treat it like a personal vault.
The default `.gitignore` already excludes `*.db` and `.env`; if you commit
this folder to a private git repo, double-check before pushing.

## CLI quick reference

```
career-hub fit --jd JOB.docx --role data --company "Acme"
career-hub apply --role data --company "Acme"
career-hub log --company "Acme" --status applied
career-hub report
career-hub scan --profile data
```

Full docs: see the upstream repository at
<https://github.com/fernandosiavato/career-hub> (or the `docs/` folder in
the source checkout).
