# applications/ — one folder per application (CLI-managed)

This folder is **managed by the CLI**. Every time you run `career-hub
apply` (or trigger the `/apply` skill in Claude Code) a new subfolder
appears here for the company you applied to. You can read these freely;
edit only the files explicitly marked as user-editable.

## Layout

```
applications/
  data/                              # subfolder per role (from config.toml)
    Acme/
      JD.md                          # the job description, normalized
      JOB.docx                       # the original JD file (if you saved one)
      keywords_gaps.md               # required keywords + gaps you have
      fit_report.md                  # fit score and breakdown
      cover_letter.md                # AI-generated, you edit before sending
      Cover_Letter_Acme_20260510.docx
      CV_Acme_<YourName>.docx        # produced by the gen_cv_acme.py script
      notes.md                       # your own notes, optional, never overwritten
```

## What is auto-generated vs what you edit

| File | Source | Safe to edit? |
|------|--------|---------------|
| `JD.md` | normalized from JOB.docx | yes (rare) |
| `keywords_gaps.md` | AI extracted from JD vs your profile | yes |
| `fit_report.md` | `career-hub fit` | regenerated on next `fit` run |
| `cover_letter.md` | AI generated | **yes — this is what you submit** |
| `CV_<Company>_<You>.docx` | generated by `cvs/scripts/gen_cv_<company>.py` | regenerate the script, do not edit the docx by hand |
| `notes.md` | you | yes |

## How an AI should help here

- **Reviewing before submitting:** ask the AI "review the cover letter
  for Acme". It should compare against `voice_and_tone.md` and the JD's
  keywords, flag any sentence that violates the user's rules, and
  suggest one edit per problem. Not a full rewrite.
- **Logging the result:** after submitting, run
  `career-hub log --company "Acme" --status applied`. The CLI updates
  the SQLite tracker. Statuses progress: `pending → applied → interview
  → technical_test → offer → hired` (or `rejected` / `withdrawn`).
- **Postmortem on rejections:** if a company rejects you, ask the AI to
  read the JD, the cover letter, and your fit_report and propose what
  was likely the gap. Add the lesson to a top-level
  `applications/_lessons.md` (you create this; the CLI never touches it).

## Privacy

Each subfolder contains your CV with full contact info and a cover
letter often referencing salary. The default `.gitignore` does not
exclude this folder; if you ever push your data dir to a remote, treat
it as private.
