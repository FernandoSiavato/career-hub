# Scanner

The scanner discovers new job openings from public ATS APIs and (optionally)
from Apify actors. It writes findings into the `scanned_jobs` table in
`jobsearch.db`. You then promote interesting ones into actual applications
using `/apply` or `career-hub apply`.

The scanner module is heavily inspired by
[career-ops](https://github.com/santifer/career-ops) — specifically the
"AI proposes, human decides" philosophy and the structured ATS
scraping. Thanks to Santi Fernandez.

## Sources

| Source | Cost | Quality | Setup |
|--------|------|---------|-------|
| Greenhouse | Free | High (full title + URL) | Just need the company's slug. |
| Ashby | Free | High | Slug. |
| Lever | Free | High | Slug. |
| Workday | Free | Medium (need a second fetch for description) | Tenant + site. |
| Apify actors | Pay-per-run | Variable (depends on the actor) | API token + actor slug. |

You can run the scanner without an Apify token. The free ATS APIs alone
cover ~80% of mid-size tech companies.

## portals.yml

Single source of truth for what to scan. Lives at the root of your data
directory. Generated by `career-hub init` with three example companies
(Anthropic, Stripe, Vercel). Edit it to match the companies you actually
follow.

```yaml
companies:
  - name: Anthropic
    ats: greenhouse
    ats_slug: anthropic
    profile: data
    enabled: true
    title_filters:
      include:
        - data
        - analyst
        - python
      exclude:
        - intern
        - principal
        - staff

  - name: Stripe
    ats: greenhouse
    ats_slug: stripe
    profile: product
    enabled: false
    title_filters:
      include:
        - product manager

  - name: SomeBigCo
    ats: workday
    workday_host: somebigco.wd1.myworkdayjobs.com
    workday_tenant: somebigco
    workday_site: External
    profile: data
    enabled: false
```

`profile` is a free-form tag that matches the `--profile` flag on
`career-hub scan`. It does **not** have to map to a role in
`config.toml`. Use it to group companies by topic ("ai", "edtech",
"climate") or by your own targeting strategy.

Where to find slugs:

| ATS | Pattern |
|-----|---------|
| Greenhouse | `https://boards.greenhouse.io/<slug>` |
| Ashby | `https://jobs.ashbyhq.com/<slug>` |
| Lever | `https://jobs.lever.co/<slug>` |
| Workday | URL contains `myworkdayjobs.com/<site>` and the tenant is the first DNS label. |

## Apify actors (optional)

```yaml
actors:
  - name: LinkedIn Jobs (data, remote)
    slug: bebity~linkedin-jobs-scraper
    profile: data
    enabled: true
    run_input:
      title: "data analyst"
      location: "Remote"
      rows: 50
```

Each run consumes Apify credits. Set `APIFY_TOKEN` in your data dir's
`.env`. The wrapper lazy-imports `apify-client`, so the package still
imports without it; you just get an "actor skipped" line in the report.

Recommended actors:

| Actor slug | Source | Notes |
|------------|--------|-------|
| `bebity~linkedin-jobs-scraper` | LinkedIn Jobs | Most reliable LinkedIn scraper. |
| `apify~web-scraper` | Custom | For one-off pages without a public API. |
| `vdrmota~indeed-scraper` | Indeed | Pricing varies. |

You can also write your own actor; the wrapper just needs a
`field_map` so it knows how to translate the actor output to the
internal `Job` schema.

## Running

```
# Scan everything enabled, free + paid sources
career-hub scan

# Free only — skip Apify
career-hub scan --no-apify

# Single company
career-hub scan --company Anthropic

# Single profile tag
career-hub scan --profile data

# Preview mode — no DB writes
career-hub scan --dry-run
```

Output is a per-source table plus a summary line:

```
Total found: 184
After title filter: 23
New (post-dedup): 11
Inserted into DB: 11
```

## Title filters

`title_filters.include` and `title_filters.exclude` are
case-insensitive substring lists. A job title passes the filter if **any**
include term matches and **no** exclude term matches.

Tips:

- Start broad and tighten as you discover false positives. Add `intern`,
  `staff`, `principal` to `exclude` once you see what slips through.
- Multiple profiles for one company: list the company twice in `companies`
  with different `profile` tags and `title_filters`.

## Browsing the results

```
career-hub scanned --profile data
career-hub scanned --status discovered --limit 50
```

Or open the optional Flask dashboard:

```
pip install career-hub[web]
career-hub web
# open http://127.0.0.1:8765
```

The dashboard lets you mark jobs as `promoted` (worth applying to) or
`rejected` with a single click and tracks reject reasons so you can see
patterns over time.

## Promoting to an application

When a scanned job looks worth applying to:

```
career-hub scanned-mark --id 42 --status promoted
```

Then either:

- Open the URL in `scanned_jobs.url`, copy the JD into a file, and run
  `career-hub apply --role data --company "Acme"`.
- Or use the `/apply` Claude Code skill: paste the JD into chat with
  the company name. The skill picks up from there.

## Maintenance

- **Workday descriptions are missing for many jobs.** Run
  `career-hub enrich-workday` to backfill.
- **`fit-scanned` lets you batch-score scanned jobs** without opening
  each one. Useful when the scanner returned 200 results and you want
  to look only at the top 20 by fit.
- **Reject patterns:** the dashboard's reject-stats endpoint surfaces
  the words that show up most in rejected job titles, which is a hint
  for new `exclude` filters.
