# cvs/ — CV templates and tailored output scripts

Two things live here:

1. **Base CV templates** as DOCX files (one per role, optionally per
   language). The `apply` and `upword` commands copy these into each
   application folder and the AI tailors them.
2. **Generated CV scripts** in `cvs/scripts/`. The `/apply` Claude Code
   skill writes one Python script per company (`gen_cv_<company>.py`) that
   builds a tailored DOCX from your profile + the JD's keywords.

## Use the question bank

When the user wants to plan a new base CV (different role, different
audience, fresh layout), open **`_template/cv-structure-template.md`**
and run it iteratively. The template plans the DOCX — it does **not**
write the DOCX itself. The user hand-edits the DOCX afterward following
the plan you produce at `cvs/_template/PLAN_<ROLE>.md`.

The template enforces three things the personalizer script depends on:
the `<COMPANY>` placeholder, the `## Highlights` heading, and the
`## Why <COMPANY>` heading. If the user's DOCX is missing those, the
plan must call it out before they save.

## Naming convention

```
cvs/
  CV_DATA.docx          # base template, English (or whatever you set as primary)
  CV_DATA_ES.docx       # Spanish version (optional)
  CV_PRODUCT.docx
  CV_PRODUCT_ES.docx
  scripts/
    gen_cv_acme.py      # generated by /apply for company "Acme"
    gen_cv_stripe.py
```

Map each role's templates in `config.toml`:

```toml
[roles.data.cv_templates]
en = "cvs/CV_DATA.docx"
es = "cvs/CV_DATA_ES.docx"
```

## How an AI should help here

- **Building a base template:** the user provides their existing CV (any
  format). Convert it to a clean DOCX preserving sections: header,
  summary, KPIs, skills, experience bullets, projects, education. Keep the
  styles simple (one or two fonts, headings as `Heading 1`/`Heading 2`)
  so `python-docx` can edit them later.
- **Reviewing a generated script:** before the user runs `gen_cv_<x>.py`,
  read the script and confirm: (a) summary references real keywords from
  the JD, (b) bullets are reordered to lead with the most relevant ones,
  (c) the title in the header matches the JD's role, (d) no skill is
  fabricated.

## Privacy

DOCX files contain your contact info, employer names, and sometimes
salary. The default `.gitignore` does not exclude DOCX, on the assumption
your data dir is private. If you ever publish your career-hub data dir,
double-check what these files contain first.
